- [x] 2. 状態管理（ScheduleStore）の実装

- [x] 2.1 ScheduleStore のコアロジック実装
  - 選択日付と時刻データを `Map<DateKey, ScheduleEntry>` で管理する状態モジュールを実装する
  - `toggleDate(date, dateKey)` — 未選択なら追加、選択済みなら削除するトグルロジックを実装する
  - `updateTime(dateKey, start, end)` — 既存エントリーの開始・終了時刻を更新するロジックを実装する
  - `subscribe(callback)` — 状態変更時に全サブスクライバーへ通知するオブザーバー機構を実装する
  - _Requirements: 1.2, 1.3, 1.4, 2.2, 3.4_

---

- [x] 3. テキスト生成ロジック（TextFormatter）の実装 (P)
  - ScheduleStore の状態を受け取り `YYYY/MM/DD(曜日) HH:MM~HH:MM` 形式テキストを返す純粋関数を実装する
  - 日付を `getDay()` で取得し `['日','月','火','水','木','金','土']` にマッピングして曜日略称を算出する
  - 複数エントリーを `dateKey`（YYYY-MM-DD）の辞書順昇順でソートして出力する
  - 終了時刻 ≤ 開始時刻のエントリーをバリデーションエラーとして出力から除外する
  - エントリーが0件の場合は空文字を返し、呼び出し元でガイダンス表示を切り替えられるようにする
  - _Requirements: 3.1, 3.2, 3.3, 3.5_

---

- [x] 4. カレンダーUI（CalendarView）の実装 (P)

- [x] 4.1 カレンダーグリッドの描画ロジック実装
  - 表示年月（`viewYear`, `viewMonth`）を内部状態として保持し、月初の曜日から空白セルを算出してグリッドを生成する
  - 各日付セルに今日・選択済み・日曜・土曜のCSSクラスを動的に付与してハイライト表示を実現する
  - `ScheduleStore.getState()` を参照して選択済み日付のハイライト状態をレンダリングに反映する
  - _Requirements: 1.1, 1.5_

- [x] 4.2 月ナビゲーションと日付選択トグルの実装
  - 前月・翌月ボタンのクリックで `viewMonth` を更新してカレンダーを再描画する
  - 日付セルのクリックで `ScheduleStore.toggleDate` を呼び出し、セルの選択状態をリアルタイムに切り替える
  - _Requirements: 1.2, 1.3, 1.6_

---

- [x] 5. 時刻ピッカーUI（TimePickerWidget）の実装 (P)

- [x] 5.1 時セレクトボックスの実装
  - `--`（未選択）と `00`〜`23` の選択肢を持つ時セレクトボックスを実装する
  - 初期値として渡された時刻文字列（`HH:MM`）があれば対応する選択肢をデフォルト選択する
  - _Requirements: 2.3, 2.5_

- [x] 5.2 分セレクトボックスとカスタム入力の実装
  - `--` / `00` / `30` / `その他` の4択セレクトボックスを実装する
  - `その他` 選択時にテキスト入力欄（`maxlength="2"`）を表示し、数字以外の文字を除去して 0〜59 の範囲で値を検証する
  - `その他` 以外の選択時はテキスト入力欄を非表示にして選択値（`"00"` or `"30"`）を使用する
  - `getValue()` メソッドを実装し、時・分のいずれかが未入力なら空文字 `""` を返す
  - _Requirements: 2.3, 2.5_

- [x] 5.3 時刻バリデーションの実装
  - 開始・終了時刻の変更のたびに「終了 > 開始」を検証し、違反時はエラーメッセージを該当行に表示する
  - バリデーション結果を `onChange` コールバックで呼び出し元（ScheduleEntryList）に通知する
  - _Requirements: 2.4_

---

- [x] 6. スケジュールエントリーリスト（ScheduleEntryList）の実装

- [x] 6.1 選択日付リストの描画とTimePickerWidget配置
  - `ScheduleStore.subscribe` で状態変化を購読し、日付選択・解除のたびにリストを再描画する
  - 選択日付を昇順ソートして、各行に `YYYY/MM/DD(曜日)` ラベルと TimePickerWidget（開始・終了）を配置する
  - 選択0件時は「カレンダーから日付を選択してください」ガイダンスを表示する
  - _Requirements: 2.1, 2.2, 2.6, 3.5_

- [x] 6.2 時刻変更のScheduleStore連携
  - 各 TimePickerWidget の `onChange` コールバックで `ScheduleStore.updateTime` を呼び出して状態を更新する
  - 日付解除時（`toggleDate` 後）に対応するエントリー行とその TimePickerWidget をDOMから除去する
  - _Requirements: 2.2, 2.6, 3.4_

---

- [x] 7. テキスト出力・コピー機能（TextOutputPanel）の実装

- [x] 7.1 生成テキスト表示エリアの実装
  - `ScheduleStore.subscribe` で状態変化を購読し、変更のたびに `TextFormatter.formatEntries` を呼び出してテキストを更新する
  - 生成テキストを等幅フォントのモノスペースエリアに表示する。エントリーが空またはバリデーションエラーのみの場合はプレースホルダーを表示する
  - _Requirements: 3.4, 3.5, 4.1_

- [x] 7.2 クリップボードコピー機能の実装
  - コピーボタンのクリックで `navigator.clipboard.writeText()` を呼び出し、生成テキストをクリップボードに書き込む
  - コピー成功時は「コピーしました！」のフィードバックを表示し、2秒後にボタンラベルを元に戻す
  - Clipboard API が利用できない環境では `Selection API`（`createRange` + `addRange`）でテキストを選択状態にして手動コピーを促すメッセージを表示する
  - 生成テキストが空（日付未選択）のときはコピー処理をスキップする
  - _Requirements: 4.2, 4.3, 4.4_

- [x] 9. 動作検証

- [x] 9.1 TextFormatterの単体動作検証
  - 単一エントリー（時刻あり・時刻なし）が正しいフォーマットで出力されることを確認する

  - 終了≤開始のエントリーが出力から除外されることを確認する

  - _Requirements: 3.1, 3.2, 3.3_

- [x] 9.2 統合シナリオの動作検証
  - 複数日付を選択し、日付ごとに異なる時刻を入力して生成テキストが正しい形式・昇順になることを確認する
  - 終了≤開始の入力でエラーが表示され、該当エントリーがテキストから除外されることを確認する
  - コピーボタンでクリップボードに正しいテキストが書き込まれ、2秒後にボタンが元に戻ることを確認する
  - 日付解除後に対応するエントリー行と生成テキストが消えることを確認する
  - _Requirements: 1.3, 2.4, 2.6, 4.2, 4.3_

- [x]* 9.3 エッジケース・フォールバックの検証
  - 日付未選択状態でコピーボタンを押した際にコピーがスキップされることを確認する
  - `その他` 分入力で 0〜59 範囲外の値が入力された場合の補正動作を確認する
  - モバイル画面幅でレイアウトが崩れないことを確認する
  - _Requirements: 3.5, 4.4_
